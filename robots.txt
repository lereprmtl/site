## 🚀 Déploiement

### Via GitHub Pages

1. **Repo** : `lereprmtl/site` (public)
2. **Branch** : `main`
3. **Settings → Pages** :
   - Source : `Deploy from a branch`
   - Branch : `main` / root
4. **Custom domain** : `lereprmtl.com` (auto-détecté via le fichier `CNAME`)
5. **Enforce HTTPS** : ✅

### DNS (Cloudflare)

Configurer chez Cloudflare pour `lereprmtl.com` :

```
A     @     185.199.108.153   DNS only
A     @     185.199.109.153   DNS only
A     @     185.199.110.153   DNS only
A     @     185.199.111.153   DNS only
CNAME www   lereprmtl.github.io   Proxied
```

GitHub Pages utilise ces 4 IPs publiques pour le custom apex domain.

⚠️ **Important** : Records A en `DNS only` (gris, non proxifié), sinon GitHub Pages ne peut pas émettre le certificat SSL Let's Encrypt automatique.

## 📝 Configuration Formspree (optionnel)

Le formulaire de liste d'attente fonctionne en deux modes :

1. **Mode par défaut** : si `FORMSPREE_ENDPOINT` n'est pas remplacé, le formulaire ouvre simplement le client courriel du visiteur avec `mailto:info@lerepr.com`.

2. **Mode actif Formspree** : pour collecter les inscriptions sans courriel utilisateur :
   - Créer un compte sur [formspree.io](https://formspree.io) (gratuit, 50 envois/mois)
   - Créer un nouveau form
   - Copier l'endpoint (ex: `https://formspree.io/f/abcdwxyz`)
   - Remplacer `FORMSPREE_ENDPOINT` dans `index.html` (deux occurrences : `action` et `const PLACEHOLDER`)
   - Commit + push
